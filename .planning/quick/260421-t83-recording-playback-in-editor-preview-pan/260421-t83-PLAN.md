## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| webview → Tauri IPC | `list_project_recordings` accepts a project ID from untrusted frontend input |
| OS filesystem → webview | `<video src>` points at a local file served by Tauri's asset protocol via `convertFileSrc` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-t83-01 | Tampering | `list_project_recordings` project-id arg | mitigate | Parse `args.id` through `uuid::Uuid::parse_str` and reject non-UUID input with `AppError::InvalidArgument` — matches existing `open_project` pattern. Only return files whose resolved extension is `mp4` (case-insensitive); do not follow symlinks outside the project's `exports/` dir. |
| T-t83-02 | Information Disclosure | Returned absolute file paths | accept | The returned `path` is already inside the user-chosen project folder; desktop app has full local filesystem scope by design (no sandboxing for this use case). No PII leaves the device. |
| T-t83-03 | Denial of Service | Directory with thousands of stale `.mp4` files | accept | Reading a dir listing + metadata for a few hundred files is sub-10ms on SSD; not a realistic DoS vector for a desktop app. If it becomes one, add a `limit` arg in a follow-up. |
| T-t83-04 | Elevation of Privilege | `<video src>` via `convertFileSrc` | mitigate | Rely on Tauri v2's asset protocol scoping (already configured for the project data dirs). Do not disable CSP. Do not pass user-controllable URLs — only paths returned by the Rust command. |
</threat_model>
