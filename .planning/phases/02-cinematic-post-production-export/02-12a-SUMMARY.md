# Phase 2 Plan 12a: Post-Production editor state + IPC foundation

**One-liner:** State + IPC scaffold for the Post-Production editor — Zustand v5 store composed of 5 slices (timeline / panels / selection / export / queue) with a localStorage-persisted panels partial and a stateless undo bridge for P13, three new Tauri commands wrapping Plan 03's repos (preset / timeline / sound_library) with T-02-36 path-escape and T-02-38 layout-size guards, and five typed IPC wrappers exposing TanStack Query keys over the Plan 10 + 11 + new commands. No React UI yet — P12b builds on this surface.

## Outcome

`UI-05` foundation: every symbol the P12b editor needs (store hooks, IPC functions, cached query keys, Tauri commands, DTOs) exists, compiles, and passes its spec tests. The editor's state contract is frozen; P12b only builds views + event handlers on top of it.

D-32 (state management) is now implemented end-to-end. D-12 (5 fixed tracks) is baked into the `tracks` shape so future code cannot add/remove tracks accidentally. D-13 (snap ON by default + 10 px threshold + Alt-disables) is exercised by three separate Vitest cases.

Tauri command surface compiles green: `cargo check -p storycapture` finishes with zero new warnings (one pre-existing `tauri-plugin-shell::Shell::open` deprecation unchanged). `pnpm --filter @storycapture/desktop typecheck` exits 0. `pnpm exec vitest run src/features/post-production/__tests__/store.test.ts` prints 7/7 green in ~220 ms.

## Deviations from Plan

### Auto-fixed issues

**1. [Rule 2 — Missing critical functionality] `.tsbuildinfo` files regenerated under `noUnusedLocals`**

- **Found during:** first `pnpm typecheck` after writing the slices.
- **Issue:** None — the build came up clean. (Noting here because the repo's `noUnusedLocals` + `noUnusedParameters` flags are strict; slices had to be written with all parameters either used or prefixed `_` from day one.) The `queue-slice` destructuring drops removed keys with `_` / `__` bindings — both intentional.

**2. [Rule 3 — Blocking] Plan's `AppState` had no project-DB handle**

- **Found during:** Task 1 Tauri command design.
- **Issue:** Plan sketch passes `state: State<'_, AppState>` to each command and assumes a project DB accessor, but `AppState` only owns `render_queue: Option<RenderQueueState>`. Adding a separate `project_db: Option<Arc<Mutex<Connection>>>` handle would create a second source of truth (gets stale when the user switches projects).
- **Fix:** Commands borrow the connection from `state.render_queue()?.db` — the render queue already holds the authoritative project DB handle. Returns `AppError::Internal("render queue not initialised")` when no project is open. Host wiring (Plan 11 / P13) plants/clears that handle on project open/close.
- **Files modified:** `apps/desktop/src-tauri/src/commands/{preset,timeline,sound_library}.rs`.
- **Commit:** `32c1088`.

**3. [Rule 2 — Missing] T-02-36 path-escape mitigation was ungated in the plan sketch**

- **Found during:** Task 1 — the plan's `<threat_model>` assigns `T-02-36` disposition `mitigate`, but the plan's command signature sketch just takes a `PathBuf` with no validation.
- **Fix:** `preset.rs::validate_preset_path` canonicalises the input and rejects paths that escape both `$HOME` and `$TMPDIR`. A unit test proves `/etc/passwd` is rejected on unix. Matches the plan's documented Threat Register mitigation exactly.
- **Commit:** `32c1088`.

**4. [Rule 2 — Missing] T-02-38 DoS guard for `timeline_save` was ungated in the plan sketch**

- Same pattern. `timeline.rs` rejects payloads > `MAX_LAYOUT_BYTES` (1 MiB) pre-persist, returning `AppError::InvalidArgument` with the exact byte count for observability.
- **Commit:** `32c1088`.

### Scope-internal choices

- **Global-scope `preset_list`/`preset_import` route to the project DB for now.** Plan 03 already mirrors the `effect_presets` schema into `app.sqlite`, but threading a second connection pool through AppState is P13 work. Scope is accepted + stored but reads/writes land in project.sqlite until the cross-project preset browser UX is built. The DTO + Rust types already know about `PresetTier::Global`, so upgrading is a pure plumbing change.
- **Queue-slice is a subscriber pattern, not a driver.** `activeJobs` is refreshed by calling `renderListActive` in a P12b `useQuery`; `applyProgress` is called inside the `streamRenderProgress` callback. P12a keeps the slice stateless wrt IPC so tests can seed it deterministically.
- **IPC wrappers don't import from `@storycapture/shared-types/ipc.ts`.** That file is regenerated on every `pnpm tauri dev`; consuming it from TS code forces a full specta codegen round-trip before `pnpm typecheck` works. Hand-rolled stable shapes match the existing `projects.ts` pattern and let P12b type-check immediately.

## Known Stubs

- **undo-bridge.ts** ships as a no-op. `dispatchUndoable(action)` executes the action immediately; `canUndo()` / `canRedo()` always return false. P13 wires the real history buffer.
- **Global-scope presets route to the project DB.** Cross-project preset browsing requires a separate app.sqlite handle on AppState. Plan 13 (settings/about) adds it when the UI grows that surface.
- **Queue-slice does not auto-refresh.** `activeJobs` starts empty; P12b calls `renderListActive` inside a TanStack Query + feeds the progress channel into `applyProgress`. P12a provides the state slot only.
- **Timeline `extraSnapTargetsMs` is plumbed but unused by the state layer.** P12b computes scene boundaries from the parsed story and passes them into `moveClip`. Tests cover the playhead + neighbour-edge paths exhaustively.

## Threat Flags

No new trust boundaries introduced beyond the plan's `<threat_model>`. The three declared mitigations are in place:

- **T-02-36 (preset_import path escape):** `validate_preset_path` in `preset.rs` canonicalises + rejects escapes. Unit test `path_guard_rejects_root_escape`.
- **T-02-37 (render queue cross-story leakage):** `render_list_active(story_id)` already filters server-side (Plan 10); P12a just surfaces the DTO.
- **T-02-38 (timeline layout DoS):** `timeline_save` rejects > 1 MiB with a descriptive `InvalidArgument` message.
