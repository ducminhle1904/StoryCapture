<objective>
Establish the typed filter-graph AST that is the single source of truth for both the WebGPU preview engine and the FFmpeg final export engine (D-01). This is the Phase 2 bet: every downstream plan (cursor, zoom, background, text, transitions, audio, export) produces AST nodes; nothing hand-writes filtergraph strings. A runtime validator enforces the canonical order (D-19, POST-08) and insta snapshot fixtures pin the emitted filter_complex strings so optimisations in later plans cannot silently reorder nodes.

Purpose: Without this foundation, Phase 2 devolves into string-soup. Pitfall #1 (label collisions) and POST-08 (canonical order) are eliminated at compile/test time, not debug time. The dual emitter (FfmpegEmit + PreviewEmit) is what makes the two-engine renderer (D-01) actually share truth.

Output: Skeleton `crates/effects` crate with AST types + builder + dual emitters + insta golden fixtures + TS mirror; no algorithm-specific logic yet (cursor/zoom/audio details land in later plans using the AST shapes defined here).
</objective>

    pub enum CanonicalStage { Source = 0, ZoomPan = 1, Background = 2, Cursor = 3, Ripple = 4, Text = 5, Transition = 6, AudioMix = 7 }

    pub fn node_stage(n: &VideoNode) -> CanonicalStage { /* pattern match each variant */ }
    pub fn audio_stage() -> CanonicalStage { CanonicalStage::AudioMix }

    pub fn validate_order(video: &[VideoNode]) -> Result<(), EffectsError> {
        let mut max_seen: Option<CanonicalStage> = None;
        for n in video {
            let s = node_stage(n);
            if let Some(prev) = max_seen { if s < prev { return Err(EffectsError::CanonicalOrderViolation(s, prev)); } }
            max_seen = Some(max_seen.map_or(s, |m| m.max(s)));
        }
        Ok(())
    }
