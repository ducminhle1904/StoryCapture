    In `sck_backend.rs`:
    1. Add enum inside the backend module:
       ```rust
       #[derive(Debug, Clone, Copy, PartialEq, Eq)]
       enum PauseState { Running, Paused, Transitioning }
       ```
    2. Add field to the backend struct: `pause_state: Arc<tokio::sync::Mutex<PauseState>>` (init `Running`).
    3. Keep existing `paused: Arc<AtomicBool>` as fast-path read signal for the frame delivery handler (do NOT remove).
    4. Rewrite `pause()`:
       ```rust
       let mut guard = self.pause_state.lock().await;
       match *guard {
           PauseState::Paused | PauseState::Transitioning => return Ok(()),
           PauseState::Running => {}
       }
       *guard = PauseState::Transitioning;
       let stream = self.stream.clone();
       tokio::task::spawn_blocking(move || stream.stop_capture()).await??;
       self.paused.store(true, Ordering::Release);
       *guard = PauseState::Paused;
       Ok(())
       ```
    5. Symmetric `resume()` — same mutex, calls `start_capture`, sets `paused=false`, state -> Running.
    6. Concurrent callers naturally await the mutex; the state-check in the match arm makes repeated pause()/resume() idempotent.
  </action>
