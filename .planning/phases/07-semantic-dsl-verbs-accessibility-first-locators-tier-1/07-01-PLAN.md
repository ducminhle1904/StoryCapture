<objective>
Extend the Story DSL parser with Tier 1 accessibility-first locator syntax while preserving 100% backwards compatibility with existing `selector`/`testid`/`aria`/bare-string forms.

Purpose: Let non-developers write demos using natural phrases (`click button "Save"`, `fill field "Email" with "..."`) that match Playwright's `getByRole`/`getByLabel`/`getByText` semantics (per D-01..D-08 in 07-CONTEXT.md).

Output: New grammar rules, three new `SelectorOrText` variants, an `AriaRole` enum, `fill` sugar desugared to `Type`, Levenshtein did-you-mean on unknown role keywords, and golden-fixture tests proving both new and legacy forms round-trip correctly.
</objective>
