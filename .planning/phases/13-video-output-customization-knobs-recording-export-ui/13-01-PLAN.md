<objective>
Extend the export IPC surface so the export modal can pass the 8 export-only knobs (Container, Codec, Rate control, HW encoder, Preset, Keyframe, Downscale, Audio) added in Phase 13. This is the ONLY backend-touching plan in Phase 13 — every other plan is frontend or persistence.

Purpose: Phase 13's `<AdvancedOutputOptions>` (Plan 13-05) needs a typed channel to ship its 8 knobs to the encoder. The current `ExportOutputDto` only has `{ format, resolution, fps, quality }` and would silently drop anything else. Per RESEARCH.md "IPC surface — Export command extension" the recommended shape is a nested `encoder_options: Option<EncoderOptionsDto>` (option b) — keeps the existing 4 fields as the "Basic" surface and groups the new 8 as the "Advanced" surface mirroring the UI structure (CD-13-01).

Output: Rust DTOs added + tauri-specta registration + auto-regen of `packages/shared-types/src/ipc.ts`. Backward-compatible: missing `encoder_options` falls through to current behavior.
</objective>
