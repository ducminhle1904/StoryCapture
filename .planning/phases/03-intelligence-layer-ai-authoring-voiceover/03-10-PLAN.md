Define in `crates/intelligence/src/tts/script.rs`:

```rust
#[derive(Debug, Clone, Serialize, Deserialize, JsonSchema)]
pub struct NarrationBatch {
    pub narrations: Vec<NarrationItem>,
}

#[derive(Debug, Clone, Serialize, Deserialize, JsonSchema)]
pub struct NarrationItem {
    pub step_id: String,
    pub text: String,
}
```

Prompt (build inline):
```
System (cached, cache_control ephemeral 5m): "You are a technical narrator. Given DSL steps, emit ONE narration per step via the `emit_narrations` tool. Rules: 
 - ≤ 80 words per narration (≤ 20s spoken)
 - Describe ONLY the action in that step; never invent UI elements or features not present
 - Match the brand tone if provided
 - Keep technical accuracy; pronounce CLI commands and brand names clearly
 - Never reference steps out of order"

User: "Story: {doc.title}\n\nSteps:\n{for step: - id={id}, verb={verb}, args={args}, label={label}}\n\nBrand tone: {brand_tone or 'neutral technical'}"
```

Implement `generate_narration_script`:
1. Build `LlmRequest` with system prompt as above, `tool_choice = {type: "tool", name: "emit_narrations"}`, `tools = [emit_narrations_tool()]`, `temperature = 0.4`, `max_tokens = 256 * story.steps.len()` (bounded).
2. Stream via provider; collect single `ToolUseComplete`.
3. Deserialize `NarrationBatch`; for each item truncate text at 80 words; compute `word_count` + `cost_estimate_usd = text.chars().count() as f64 * 0.30 / 1000.0` (ElevenLabs rate).
4. Verify each `step_id` from the batch exists in `story.steps`; drop any unknown IDs (log warning); for any story step missing from batch, synthesize a fallback narration `"{step.label}"` (degraded but non-empty).

Register `pub mod script;` in `crates/intelligence/src/tts/mod.rs`.
