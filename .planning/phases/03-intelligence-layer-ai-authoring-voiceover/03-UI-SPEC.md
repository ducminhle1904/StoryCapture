## Typography

Four roles, two weights (Regular 400 + Semibold 600). No italics. No Display size in Phase 3 (no hero/marketing surface).

| Role | Size | Weight | Line Height | Used For |
|------|------|--------|-------------|----------|

| Body | 14px | 400 | 1.5 (21px) | Chat bubble text, settings row description, voice card preset name, dry-run summary text, popover body copy |

## Color

Inherits Phase 2 dark-first palette (D-23 Runway/Linear/ElevenLabs inspiration). Tokens below live in `@theme` as CSS variables; hex values are the resolved dark-mode defaults. Light-mode equivalents follow the same 60/30/10 logic with surface inversion — planner to mirror via Tailwind v4 `@theme` variants.

| Role | Token | Dark Hex | Usage |
|------|-------|----------|-------|
| Dominant (60%) | `--background` | `#0A0B0F` | App background, chat panel backdrop, editor backdrop |
| Secondary (30%) | `--card` / `--surface-raised` | `#13151C` | Chat bubbles (both roles, disambiguated by border), diff cards, voice preset cards, settings rows, dry-run summary panel, hover popover backdrop |
| Border | `--border` | `#242733` | Card outlines, chat bubble borders, panel dividers, gutter column separator |
| Foreground | `--foreground` | `#E6E8EE` | Body text, Emphasis text |
| Muted foreground | `--muted-foreground` | `#8A90A2` | Labels, meta (timestamp, token counts, locale flags), placeholder copy |
| Accent (10%) | `--accent` (violet gradient `#7C3AED → #A855F7`) | `#7C3AED` | See reserved list below |
| Destructive | `--destructive` | `#E5484D` | Grammar error gutter marker, reject button (✕) on diff cards, invalid API key status, hard-fail dry-run step, delete-conversation confirmation CTA |
| Success | `--success` | `#30A46C` | Approve button (✓) on diff cards, valid API key status, pass dry-run step |
| Warning | `--warning` | `#E5A000` | Selector-fallback gutter marker (⚠️), rate-limit banner, token counter when session spend >$1.00 |
| Info | `--info` | `#4493F8` | Semantic LSP warning gutter marker (undefined verb, arg mismatch), dry-run queued/running state |

### Accent reserved for (10% rule — never "all interactive elements")

1. **Primary CTA on NL chat composer** ("Send", keyboard `⌘↵`)
2. **"Generate script" button** inside TTS per-step script editor
3. **"Approve all" affordance** at top of chat diff review (bulk accept)
4. **"Test connection" success pulse** when API key validates (one-shot animation, 400ms)
5. **Streaming token indicator** (shimmering gradient dot next to assistant bubble while tokens stream)
6. **Voice preset "Featured" badge** (max 2 of 8 bundled presets)
7. **Dry-run "Start" primary button** (resting state; running state switches to warning)

All other interactive elements (secondary buttons, per-step ✓/✎/↻/✕ actions, voice card Preview, settings fields) use `--foreground` / `--border` / semantic colors — **never** accent.

### Contrast (WCAG 2.1 AA, UI-10)

- Foreground on dominant: `#E6E8EE` on `#0A0B0F` → 16.8:1 (AAA).
- Muted on secondary: `#8A90A2` on `#13151C` → 5.2:1 (AA for normal text).
- Destructive red on secondary: `#E5484D` on `#13151C` → 4.7:1 (AA).
- Success green on secondary: `#30A46C` on `#13151C` → 4.6:1 (AA).
- Warning amber on secondary: `#E5A000` on `#13151C` → 6.1:1 (AA).
- Diff red/green must meet 4.5:1 on both inline-diff surface (red `#5C1D1F` bg + `#FF8A8F` fg; green `#0E3A22` bg + `#78DDA4` fg) — tuned to preserve Screen Studio-style visibility.

## Motion Contract (motion/react)

Phase 3 motions are micro-interactions only; no scroll-linked or page-transition animations. All motion respects `prefers-reduced-motion` (WCAG 2.3.3).

| Event | Duration | Easing | Element |
|-------|----------|--------|---------|
| Chat bubble enter | 180ms | `easeOut` | Translate Y 8px → 0, opacity 0 → 1 |
| Streaming dot pulse | 900ms loop | `easeInOut` | Opacity 0.4 ↔ 1.0 |
| Diff card expand/collapse | 220ms | `easeInOut` | Height auto-measure |
| Diff card approve success | 260ms | `easeOut` | Card border tints success, then fades 400ms and removes |
| Gutter status transition (queued→running→pass) | 160ms | `easeInOut` | Background color interpolate |
| Voice preview playing ring | Continuous rotation 2.4s linear | `linear` | Conic-gradient ring around voice card avatar |
| Toast (sonner) | shadcn default | — | Do not override |
| Popover/hover-card | 120ms | `easeOut` | Fade + scale 0.96 → 1.0 |
| Panel collapse/expand | 200ms | `easeInOut` | Width interpolate |

Reduced-motion fallback: all motions collapse to instant state changes except streaming dot (which becomes a static filled dot).

## Accessibility Contract (WCAG 2.1 AA — UI-10)

### Keyboard navigation

- **Chat panel:** `⌘1` focuses composer; `Tab` walks pending diff cards; within card, `A`/`E`/`R`/`Backspace` trigger approve/edit/regen/reject; `Esc` closes card's expanded state; `⌘⇧A` bulk approve.
- **Editor ↔ chat swap:** `⌘2` focuses editor from chat; `⌘1` focuses chat from editor. Never trap focus.
- **Voice catalog dialog:** `Tab` cycles filters → grid → close; `↑↓←→` navigates grid; `Space` previews; `↵` selects; `Esc` closes.
- **Dry-Run panel:** focusable; `↑↓` navigates step rows; `↵` jumps editor cursor to that step line.
- **LSP hover popover:** triggered by `Tab` onto gutter icon (not just hover); `↵` opens popover, `Esc` closes, focus returns to gutter icon.

### Focus management

- All modal dialogs trap focus (Base UI `Dialog` default).
- Opening chat panel does NOT steal focus from editor unless user invoked via CTA.
- Focus ring: 2px outer ring `--accent` with 2px offset, visible on dark + light modes. Never remove `:focus-visible`.

### Screen reader labels

- Gutter markers: `aria-label="Lỗi ngữ pháp ở dòng {N}"` / `"Cảnh báo selector ở dòng {N}"` / `"Gợi ý ngữ nghĩa ở dòng {N}"`.
- Diff card action buttons: `aria-label="Chấp nhận bước {N}"` etc.
- Voice preview button: `aria-label="Nghe thử giọng {voice_name}, ngôn ngữ {locale}"`; during playback: `aria-pressed="true"`.
- Streaming state: `<div role="status" aria-live="polite">Đang sinh bước {N}…</div>`.
- Token counter: `aria-label="Chi tiêu session: ${cost} USD, {tokens} token. Nhấn để xem chi tiết"`.
- LSP hover popover: `role="tooltip"` with `aria-describedby` linkage.
- All icon-only buttons carry `aria-label` (matches CLAUDE.md WCAG baseline).

### Color-independence

- Gutter markers use color AND shape (● error, ⚠ warning, ℹ info). Never rely on color alone.
- Diff red/green lines carry `-` / `+` prefix per convention.
- Dry-run status uses badge text (`Queued` / `Running` / `Pass` / `Fail`) alongside color.
