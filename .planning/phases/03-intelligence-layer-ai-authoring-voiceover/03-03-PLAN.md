<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Webview → Tauri IPC | User enters API key string in webview; crosses into Rust via `key_set` |
| Rust core → OS keychain | Key leaves process to OS-managed secure store |
| Rust core → provider HTTPS | Key crosses network boundary only over rustls TLS |
| Rust core → tracing/log | Key must NEVER cross this boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Information Disclosure | `#[tracing::instrument]` auto-debug | mitigate | `skip(app, key)` on `key_set`; `Redacted<String>` wrapper inside `key_test`; redaction layer from Plan 01 as defence-in-depth; G1-extension test `key_no_leak_tests` enforces |
| T-03-03-02 | Information Disclosure | `KeyTestReport.detail` | mitigate | `detail` derived from HTTP status code + reason phrase only; NEVER includes request headers; unit test greps the returned detail for key substring |
| T-03-03-03 | Spoofing | Provider probe TLS | mitigate | `reqwest` built with `default-features = false, features = ["rustls-tls"]` — forces rustls; no plaintext HTTP allowed (rejects `http://` scheme by constant URL) |
| T-03-03-04 | Elevation of Privilege | Keychain access on macOS | accept | `tauri-plugin-keyring` uses OS-level ACL (Security framework Keychain). macOS may prompt for consent; acceptable UX per CONTEXT "API key onboarding". No mitigation needed. |
| T-03-03-05 | Tampering | Webview injecting bogus provider enum | mitigate | `ProviderId` is a closed Rust enum with `#[serde(rename_all = "snake_case")]`; serde rejects unknown variants, returning validation error before any keychain access |
| T-03-03-06 | Repudiation | Key save/delete without audit trail | accept | Solo-user desktop app; OS keychain itself logs access. No additional audit table in Phase 3. |
</threat_model>
