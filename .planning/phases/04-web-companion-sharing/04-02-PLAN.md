       - `import NextAuth from 'next-auth'`
       - Providers: `GitHub` and `Google` (import from `next-auth/providers/github` and `next-auth/providers/google`)
       - Adapter: `PrismaAdapter(prisma)` from `@auth/prisma-adapter`

         ```
         events: {
           createUser: async ({ user }) => {
             await prisma.workspace.create({
               data: {
                 name: 'Personal',
                 slug: `personal-${user.id}`,
                 isPersonal: true,
                 members: { create: { userId: user.id, role: 'OWNER' } },
               },
             });
           },
         }
         ```

       - `mintDesktopToken(userId: string)`: creates a long-lived token (30 days) signed with JWT_SECRET

       - `mintJwt(userId: string)`: creates a short-lived JWT (15 min per D-07) for SSE subscriptions

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> NextAuth | OAuth callback with code/state crosses from provider |
| Desktop -> /api/auth/desktop-token | Desktop exchanges session for long-lived token |
| Client -> protectedProcedure | Auth header/session cookie crosses into tRPC |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | S (Spoofing) | OAuth flow | mitigate | NextAuth handles PKCE + CSRF state param automatically |
| T-04-05 | E (Elevation) | protectedProcedure | mitigate | Every protected tRPC procedure checks session via middleware; throws UNAUTHORIZED |
| T-04-06 | I (Info Disclosure) | Desktop token | mitigate | Token stored in OS keychain (desktop side); transmitted only over HTTPS |
| T-04-07 | T (Tampering) | JWT | mitigate | JWTs signed with HS256 + JWT_SECRET; short expiry (15 min) for SSE tokens |
| T-04-08 | R (Repudiation) | desktop-token endpoint | mitigate | Validates session exists in DB before issuing token; session is revocable |
</threat_model>
