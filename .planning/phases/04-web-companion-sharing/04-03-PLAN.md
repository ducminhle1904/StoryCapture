  <action>
    1. Add `tauri-plugin-oauth` to Cargo.toml if not already present (for localhost redirect server). If the plugin is not available or too complex, implement a minimal localhost HTTP server using `tokio::net::TcpListener` that captures the OAuth callback — the key functionality is: spawn server on a random port, open browser to web companion OAuth URL with that port as callback, capture the redirect with auth code/session token, shut down the server.

    2. Create commands/web_account.rs with these Tauri commands:

       `start_web_oauth`:
       - Spawns a temporary localhost HTTP server on a random available port
       - Opens system browser to `{WEB_COMPANION_URL}/api/auth/signin/github?callbackUrl=http://localhost:{port}/callback`
       - The WEB_COMPANION_URL should default to `https://storycapture.app` but be configurable via app settings or env var for development
       - Returns the port number to the frontend

       `complete_web_oauth`:
       - Waits for the localhost server to receive the callback (with timeout)
       - Extracts the session token from the callback
       - Calls the web companion's `/api/auth/desktop-token` endpoint to exchange for a long-lived API token
       - Stores the API token in OS keychain via tauri-plugin-keyring with key `web_api_token`
       - Also fetches and stores basic user info (email, name, avatar URL) in keychain with key `web_account_info`
       - Returns the account info to the frontend

       `get_web_account`:
       - Reads `web_account_info` from keychain
       - Returns `Option<WebAccountInfo>` with { email, name, avatarUrl, connectedAt }

       `disconnect_web_account`:
       - Deletes `web_api_token` and `web_account_info` from keychain
       - Returns success

       `get_web_api_token`:
       - Reads `web_api_token` from keychain (used by upload + sync commands in later plans)
       - Returns `Option<String>`

    3. Register all commands in commands/mod.rs and wire into the Tauri app builder.

    Error handling: Use the existing `AppError` pattern with `From` impls. Return structured errors for: no internet, OAuth timeout (30s), token exchange failure, keychain access failure.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Desktop -> localhost server | OAuth callback with session data |
| Desktop -> Web API | Token exchange over HTTPS |
| Keychain -> Desktop app | Secret retrieval from OS keychain |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | S (Spoofing) | localhost OAuth | mitigate | Random port + short-lived server (30s timeout) + verify callback origin |
| T-04-10 | I (Info Disclosure) | API token | mitigate | Stored in OS keychain via tauri-plugin-keyring; never in SQLite or plaintext (per AI-05 pattern) |
| T-04-11 | D (DoS) | OAuth timeout | mitigate | 30-second timeout on localhost server; clean shutdown on timeout |
</threat_model>
