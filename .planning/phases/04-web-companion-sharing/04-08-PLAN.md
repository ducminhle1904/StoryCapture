    1. Create lib/geo.ts per RESEARCH.md Analytics Architecture:
       - Singleton MaxMind GeoLite2 reader using `@maxmind/geoip2-node`
       - `Reader.open()` on first call, cache the reader instance
       - Read from `public/geolite2/GeoLite2-Country.mmdb` (gitignored, downloaded on deploy)
       - `getCountry(ip: string): string` — returns ISO 3166-1 alpha-2 code or 'XX' for unknown
       - Graceful fallback: if MMDB file missing or lookup fails, return 'XX'
       - Add `import 'server-only'`

    2. Create scripts/download-geolite2.sh:
       - Downloads GeoLite2-Country.mmdb from MaxMind using MAXMIND_LICENSE_KEY env var
       - Extracts to public/geolite2/
       - Intended to run during Vercel build or CI deploy
       - `curl "https://download.maxmind.com/app/geoip_download?edition_id=GeoLite2-Country&license_key=${MAXMIND_LICENSE_KEY}&suffix=tar.gz" | tar xz --strip-components=1 -C public/geolite2/`

    3. Create app/api/analytics/session/route.ts:
       - GET endpoint that sets a session-ID cookie if not present
       - Cookie: httpOnly, secure, SameSite=Lax, 30-day expiry, name: `sc_session`
       - Value: random UUID
       - Per D-06: no PII, no fingerprinting, first-party only
       - Returns `{ sessionId }` for the viewer page to use

    4. Create app/api/analytics/ingest/route.ts per RESEARCH.md example:
       - POST endpoint accepting: `{ videoId, event, timestamp, sessionId, currentScene?, watchDurationSec? }`
       - Validate event is one of: 'play', 'pause', 'seek', 'scene_enter', 'ended'
       - Resolve country from IP via `getCountry(req.headers.get('x-forwarded-for'))`
       - Insert ViewEvent into database
       - Return `{ ok: true }`
       - No auth required (public endpoint — viewers are anonymous)
       - Rate limiting: basic in-memory per-IP throttle (10 events/sec max) to prevent abuse. Simple Map-based, not production-grade.

    5. Create trpc/routers/analytics.ts:

       `dashboard` (protectedProcedure, query):
       - Input: `{ videoId, days?: number }` (default 30 per D-06)
       - Verify user is editor/owner of the video's workspace
       - Compute from ViewEvent table:
         - totalPlays: COUNT where event='play'
         - uniquePlays: COUNT(DISTINCT sessionId) where event='play'
         - avgDurationSec: AVG(watchDurationSec) where event='ended'
         - medianDurationSec: approximate median via percentile query or sorted array
         - sceneDropoffs: for each scene, count scene_enter events; drop-off = scene_enter[N] - scene_enter[N+1]
         - countryBreakdown: GROUP BY country, COUNT for play events
       - Return aggregated data

       `dailyStats` (protectedProcedure, query):
       - Input: `{ videoId, days?: number }`
       - Return DailyVideoStats rows for the period (for chart display)
       - If no pre-aggregated rows, compute on-the-fly from ViewEvent

       `aggregateDaily` (protectedProcedure, mutation):
       - Input: `{ videoId, date }` (manual trigger for daily rollup)
       - Compute stats for a given day and upsert into DailyVideoStats
       - In v1, this can be triggered manually or via a simple cron. Full scheduled aggregation is a deployment concern.

    6. Update lib/constants.ts with analytics constants: `ANALYTICS_EVENTS`, `MAX_RETENTION_DAYS: 90`, `DASHBOARD_DAYS: 30`

    7. Merge analyticsRouter into _app.ts

    8. Create apps/web/vercel.json with a cron declaration for analytics aggregation (per D-06 near-real-time):
       ```json
       {
         "crons": [{
           "path": "/api/cron/aggregate-analytics",
           "schedule": "* * * * *"
         }]
       }
       ```
       Note: Vercel minimum cron interval is 1 minute. This satisfies D-06's "every 30s" as closely as possible on Vercel.

    9. Create apps/web/src/app/api/cron/aggregate-analytics/route.ts:
       - GET endpoint that Vercel cron triggers every minute
       - Verify the request is from Vercel cron (check `Authorization: Bearer ${CRON_SECRET}` header)
       - Query all videos with ViewEvents in the last 2 minutes that don't have a DailyVideoStats entry for today
       - For each, compute and upsert DailyVideoStats (total plays, unique plays, avg/median duration, country breakdown, scene dropoffs)
       - Also clean up raw ViewEvent records older than 90 days (per D-06 retention policy)
       - Return `{ aggregated: N, cleaned: M }`
