LivePreview status machine (new internal state):
```ts
type PreviewStatus = 'idle' | 'starting' | 'running' | 'recovering' | 'unavailable';
// Transitions:
//   idle → starting (startPreviewStream invoked)
//   starting → running (first frame received)
//   starting/running → recovering (listen error OR startPreviewStream rejected with transient error)
//   recovering → running (retry succeeded)
//   recovering → unavailable (retry failed)
//   any → unavailable (UnavailableOnBackend from backend)
```

Saturation counters (internal, logged):
- Sidecar: `state.previewDropCount` incremented when `state.latestFrame` is overwritten before flush.
- Rust pump: local counter in the spawned task; emitted via `tracing::warn!` every 30 seconds if non-zero, then reset.
- Webview: local counter in LivePreview; decode-drop on bitmap replacement logged to console at debug level, and to a tracing event every 30s via a tauri command if non-trivial.

everyNthFrame selection (sidecar):
```js
// At startPreviewStream, inspect state.page viewport / deviceScaleFactor.
// If devicePixelRatio >= 2 OR launch viewport width > 1600 → state.previewEveryNth = 2
// Otherwise → state.previewEveryNth = 1
```

       ```ts
       async function startWithRetry(retriesLeft: number) {
         try {
           await startPreviewStream();
           setStatus('running');
         } catch (err) {
           if (isUnavailableOnBackend(err)) {
             setStatus('unavailable');
             return;
           }
           if (retriesLeft > 0) {

           setStatus('unavailable');

       }
       ```
