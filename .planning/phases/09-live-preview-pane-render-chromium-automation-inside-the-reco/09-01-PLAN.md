```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct PreviewFrame {
    pub data: String,      // base64 JPEG
    pub width: u32,
    pub height: u32,
    pub timestamp: f64,
}

#[derive(Debug, serde::Deserialize)]
#[serde(untagged)]
enum SidecarMsg {
    Response(JsonRpcResponse),                                // existing shape with id
    Notification { method: String, params: serde_json::Value } // no id
}

impl PlaywrightSidecarDriver {
    pub fn subscribe_preview(&self) -> tokio::sync::watch::Receiver<Option<PreviewFrame>>;
    pub async fn call_preview_start(&self) -> Result<(), AutomationError>;
    pub async fn call_preview_stop(&self)  -> Result<(), AutomationError>;
}
```

CDP params (authoritative, from 09-RESEARCH.md):
- `Page.startScreencast` params: `{ format: "jpeg", quality: 80, maxWidth: 1280, maxHeight: 720, everyNthFrame: 1 }`
- `Page.screencastFrameAck` params: `{ sessionId: <frame.sessionId> }`
- Must ack latest sessionId within same CDP handler; skipping acks on dropped older frames is safe (ack is cumulative).

Backpressure contract: exactly ONE pending frame per layer.
- Sidecar: `state.latestFrame` single slot + `setImmediate` flusher.
- Rust: `tokio::sync::watch::channel::<Option<PreviewFrame>>(None)` (latest-wins by design).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Chromium renderer → sidecar CDP session | Renderer-controlled content produces screencast frames; sidecar receives untrusted base64 payloads |
| Sidecar → Rust (stdio JSON lines) | New notification shape; parsing must be fault-tolerant (malformed JSON cannot crash reader) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01-01 | Tampering | Sidecar stdout line parser | mitigate | Untagged SidecarMsg + warn-log on serde error; reader loop never panics (Task 2). |
| T-09-01-02 | DoS | Screencast frame flood | mitigate | `maxWidth=1280`, `maxHeight=720`, latest-wins single-slot at sidecar AND Rust; Chromium self-throttles via ack policy (Task 1 + Task 2). |
| T-09-01-03 | Information Disclosure | Preview frame bytes in logs | mitigate | Never log `frame.data` (base64 JPEG). Only log `width/height/timestamp` on warn paths. |
| T-09-01-04 | DoS | Huge malformed `params.data` OOM | accept | Base64 string inflation capped implicitly by Chromium's maxWidth/maxHeight; no explicit length check needed for cosmetic-only path. Acceptable for v1. |
</threat_model>
