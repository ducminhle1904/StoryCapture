<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend (React) → Tauri IPC → Rust capture crate | User/UI-supplied `CaptureTarget` (including `WindowId` / pid) crosses process-privilege boundary; macOS capture has elevated permission (TCC Screen Recording) that must not be abusable by a tampered renderer |
| Rust capture → macOS ScreenCaptureKit | OS-enforced TCC boundary; we must not attempt to circumvent denied grants |
| SCShareableContent → logs / telemetry | Window titles may contain PII (bank tabs, medical apps, private URLs); StoryCapture has a zero-telemetry constraint but logs also leak locally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering / EoP | `start_capture(target)` IPC command | mitigate | Validate `WindowId` against the set returned by the most recent `list_windows()` call; refuse unknown ids with typed error. Applied in Task 3. |
| T-05-01-02 | InfoDisclosure | `SCWindow::title()` values in logs | mitigate | Log titles ONLY at TRACE level; default log level is INFO; never serialize titles into crash reports or `tauri-plugin-log` uploads. Applied in Task 1 (window.rs) and Task 3 (IPC command). |
| T-05-01-03 | EoP | Cross-user window capture via pid spoofing | mitigate | `SCShareableContent` is session-scoped by the OS — documented in RESEARCH.md Security Domain. Additional guard: refuse if `SCRunningApplication.process_id` resolves to a uid different from `getuid()` (belt-and-braces, cheap). Applied in Task 1. |
| T-05-01-04 | InfoDisclosure | Capture of hidden/system windows | mitigate | Filter `window_layer == 0` + `is_on_screen`; the filter is in `list_windows()` and also re-applied at `start_capture` validation time. Applied in Task 1 + Task 3. |
| T-05-01-05 | T/I | TCC revoked mid-session | mitigate | SCStreamDelegate `didStopWithError` → `CaptureEvent::BackendFailed` → pipeline stops + encoder finalizes. D-03. Applied in Task 2. |
| T-05-01-06 | S / E | "Reset permission" UX tempting a bypass attempt | accept | We only surface `open x-apple.systempreferences:com.apple.preference.security`; we never call `tccutil` ourselves. D-08. |
| T-05-01-07 | D (DoS) | Malicious renderer spams `start_capture` | accept | Low-value target on a single-user desktop; rate-limited implicitly by single active pipeline. Noted; no mitigation this phase. |
</threat_model>
