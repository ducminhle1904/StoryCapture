<objective>
Ship three capture-config knobs in one coherent slice: (a) per-display region capture with platform-correct crop (SCK source_rect on macOS, CPU crop on Windows — honoring the RESEARCH amendment to D-07 that `windows-capture 2.0.0` has NO native region API), (b) Chromium chrome-hiding via `--app=<url>` launch arg, (c) per-recording cursor toggle (non-sticky, default ON).

Purpose: The polish layer that makes recordings look intentional — a cropped demo window, no browser chrome noise, cursor decisions per story. Each of these three features is a UI toggle + a capture/launch-config knob; they share the recorder surface and don't collide in files, so they ship together.

Output: `CaptureTarget::DisplayRegion` enum variant; SCK source-rect wiring on macOS; WGC CPU-crop path on Windows (with bench); region-selection overlay window; `LaunchConfig.args` through to Playwright sidecar; cursor + chrome-hiding toggles in recorder.

**⚠ D-07 amendment flag for plan-checker:** CONTEXT.md D-07 says "Windows equivalent via WGC crop rect. No custom FFmpeg crop filter — let the platform API do it." RESEARCH.md verified this is impossible with `windows-capture = 2.0.0` (`DirtyRegionSettings` is an update-optimization hint, not a crop). We implement post-capture CPU crop on Windows, matching the "no FFmpeg crop filter" constraint — the spirit of D-07 is preserved (platform code crops, not FFmpeg) while the letter (WGC native crop API) is not available. Plan-checker should validate this amendment.
</objective>
