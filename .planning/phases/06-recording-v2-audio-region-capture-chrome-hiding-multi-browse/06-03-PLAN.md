<objective>
Ship two independent-but-recorder-UI-adjacent features in one plan: (a) Multi-browser auto-follow — replace the hardcoded `title_hint: "Chromium"` with a preset-driven map so Edge, Brave, Chrome channels all auto-follow correctly; (b) Live preview thumbnail — a static 320×200 image of the selected target, refreshed every 2s, rendered between Target dropdown and Start button, backed by macOS `SCScreenshotManager` and Windows single-frame WGC capture.

Purpose: Multi-browser makes auto-follow work for the 30%+ of users who don't use Chrome. Live preview tells users "yes, that's the window you think it is" before they commit to recording — it resolves in 2–3 seconds of view time and is the single cheapest polish win in the phase.

Output: `BROWSER_TITLE_HINTS` map + wiring into Playwright auto-follow, `capture_target_thumbnail` IPC command with platform impls, `TargetThumbnail` React component.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI → BROWSER_TITLE_HINTS | Preset key from BrowserRow (settings-sourced; user-controlled but user is the threat actor) |
| Tauri command → SCScreenshotManager / WGC | Capture target crosses into OS-level kernel capture API |
| Thumbnail bytes → UI | PNG bytes flow into React img src (data URL) |
| Logs → disk | title_hint could leak window titles into log files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-15 | Information Disclosure | title_hint leaks window titles into log files | mitigate | Truncate title_hint to first 40 chars when logging at INFO level; DEBUG level also truncates (reuses Phase 5 D-07 redaction). No full-title strings in logs. Tracing span field marked with `#[field(display = ...)]`. |
| T-06-16 | Denial of Service | Thumbnail refresh fires every 2s even when window hidden or minimized, wasting CPU/battery | mitigate | `useQuery({ enabled: !isRecording && isVisible })` gated by IntersectionObserver + visibilitychange listener. Also stop when recorder view navigates away. |
| T-06-17 | Information Disclosure | Thumbnail of an offscreen window could expose sensitive content (e.g., banking tab the user forgot about) | accept | User selected the target; seeing the thumbnail is a feature, not a leak. UX is consent. No network transmission. |
| T-06-18 | Denial of Service | Thumbnail capture racy with TCC permission revocation mid-refresh | mitigate | capture_thumbnail returns typed error on TCC denied; React query treats as error state → placeholder; no retry storm (default React Query behavior bounds retries to 3). |
| T-06-19 | Tampering | Malicious preset string in BrowserRow could cause lookup of unmapped key | accept | `titleHintFor` returns undefined for unknown keys; no code injection surface — string flows to string comparison in find_window_by_pid. Tested (Task 0 Vitest). |
| T-06-20 | Denial of Service | Thumbnail data URL memory leak on target change | mitigate | Task 3 explicitly revokes the previous objectURL on target change via useEffect cleanup. Verified in Vitest with createObjectURL/revokeObjectURL mock assertions. |
| T-06-21 | Information Disclosure | Thumbnail bytes stored in React Query cache persist longer than needed | mitigate | Query `cacheTime: 0` — no cache beyond active refetch cycle. |
</threat_model>
