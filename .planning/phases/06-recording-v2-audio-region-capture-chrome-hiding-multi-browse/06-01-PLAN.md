<objective>
Ship microphone audio capture wired through FFmpeg mux, honoring every locked Audio decision (D-01..D-04) while working around both blocking constraints surfaced by RESEARCH.md: (a) cpal#970 — Windows WASAPI callback deregisters on cross-thread sends (use ringbuf + std drain thread), and (b) Tauri sidecar cannot expose fd ≥ 3 (use named pipe, not `pipe:3`).

Purpose: Adds voiceover to recordings — the single highest-value polish item from Phase 5's deferred list, and the piece with the most architectural risk. Landing it first de-risks the rest of Phase 6.

Output: New `crates/capture/src/audio/` module, FFmpeg arg extension in `crates/encoder`, Tauri command + UI picker, NSMicrophoneUsageDescription shipped in the bundle.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI → Tauri command | `audio_device_id` string from untrusted React state |
| Tauri → cpal | Device name enumerated server-side, passed back in — echo path |
| cpal callback → drain thread | Lock-free ringbuf; no user data flows here |
| FFmpeg ← named pipe | Raw f32 samples from mic; no parsing; opaque byte stream |
| OS → mic hardware | TCC-gated; user must grant explicitly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Mic eavesdropping (captures voice when user didn't intend) | mitigate | Per-recording explicit opt-in (D-02); non-sticky picker (Task 4); default = "No audio"; NSMicrophoneUsageDescription is honest about trigger. No background mic activation path exists in code. |
| T-06-02 | Denial of Service | Ringbuf unbounded growth if drain thread stalls | mitigate | Fixed 2-second HeapRb capacity (`sample_rate * channels * 2`). `push_slice` returns short count on full — samples are dropped, not buffered. No OOM path. |
| T-06-03 | Tampering | `audio_device_id` from frontend could be crafted to target a device user hasn't opted into (still mic, but a specific one) | accept | TCC grant is per-app not per-device; any input device the user has plugged in is already within the mic-permission scope they granted. No privilege boundary crossed. |
| T-06-04 | Denial of Service | Named-pipe permissions too loose (other local users could connect) | mitigate | Unix: mkfifo with mode `0o600` (owner RW only). Windows: CreateNamedPipeW with default security descriptor limits to same session. Path includes a fresh UUID so collisions are prevented. |
| T-06-05 | Elevation of Privilege | cpal callback injection | accept | cpal callback executes in OS audio thread; no user input flows in; Rust type system prevents arbitrary code execution. |
