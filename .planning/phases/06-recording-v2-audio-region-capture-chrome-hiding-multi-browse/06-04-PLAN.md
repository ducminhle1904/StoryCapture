  <action>
    Create `tests/windows_real_capture_e2e.rs` with two `#[ignore]` `#[tokio::test]` functions, both gated by `#[cfg(all(windows, feature = "real-capture-windows"))]`. Each test:
    (1) Spawns Playwright via the Phase 5 `tools/e2e-playwright-capture` binary or direct automation crate call — reuse the existing harness don't reinvent. Preset = 'chromium' (bundled Playwright browser, always available).

    (5) Deletes temp file on success, leaves on failure for operator inspection.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Self-hosted runner → repo | Runner executes arbitrary PR code on each `workflow_dispatch` trigger |
| Workflow → workspace | Writes MP4 artifacts readable by anyone with repo access |
| Operator → runbook | Operator executes documented commands on their Windows host |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-22 | Elevation of Privilege | Self-hosted runner compromise via PR workflow trigger | mitigate | Workflow is `workflow_dispatch`-only (no `pull_request` trigger) — only users with write access to the repo can dispatch. No secrets referenced in workflow. Runner is ephemeral-restart-recommended in the operator docs (comment in the YAML header). |
| T-06-23 | Information Disclosure | MP4 artifact could contain sensitive content captured during test | accept | Tests capture Playwright-launched Chromium loading a blank or test URL — no user data. Artifacts retained 7 days only. |
| T-06-24 | Tampering | Operator running manual script on untrusted Windows host could be handed a malicious build | accept | Operator controls the host; building from a known git ref is operator responsibility. Runbook says "git clone repo" so provenance is explicit. |
| T-06-25 | Denial of Service | Runner availability blocks critical releases | mitigate | D-23 + Task 2 manual script explicitly provide the fallback path — releases never blocked on runner. SUMMARY documents which path was taken. |
| T-06-26 | Repudiation | Manual-path evidence could be fabricated | accept | ffprobe output + MP4 attached to SUMMARY provide reasonable evidence; solo-developer workflow means the auditor and the actor are the same person. No multi-party trust model. |
| T-06-27 | Elevation of Privilege | Runner with auto-login session could be SSH'd or RDP'd into externally | mitigate | Operator runbook instructs: use a dedicated VM, not a shared dev machine; firewall inbound; disable RDP unless actively debugging. Documented in capture-windows-e2e.yml comment block. |
</threat_model>
