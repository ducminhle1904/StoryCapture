<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebView → Tauri host | Untrusted JS can only call commands declared in capabilities manifest; all inputs validated at command entry. |
| Command handler → keyring | API keys travel through keyring plugin; never returned to renderer except via explicit `load_secret` command. |
| Panic hook → renderer | Panic payload sanitized (no PII, no keychain values) before emit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Arbitrary JS invoking privileged commands | mitigate | Tauri v2 capability manifest `capabilities/default.json` whitelists only the main window; plugin permissions scoped per command. |
| T-03-02 | Information Disclosure | Keychain secrets in logs | mitigate | `store_secret`/`load_secret` never log the `value` parameter; `tracing` uses `#[instrument(skip(value))]`. |
| T-03-03 | Tampering | Renderer tampering with error messages | accept | Errors flow server→client only; client can display but not persist back. |
| T-03-04 | Denial of Service | Unhandled panic kills process silently | mitigate | `panic_hook.rs` catches, logs, emits event, chains to default; renderer shows modal. |
| T-03-05 | Elevation of Privilege | Plugin permission drift | mitigate | `capabilities/default.json` explicit allow-list; CI `cargo check` fails if plugin not registered. |
| T-03-06 | Repudiation | Crash without diagnostics | mitigate | Tracing + file rotation + panic backtrace captured to disk before modal appears. |
</threat_model>
