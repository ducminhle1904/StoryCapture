<objective>
**Wave 0 hard release gate.** Build universal static FFmpeg 7.x binaries (LGPL-only, no nested dylibs) for all three target triples, then prove the **entire macOS signing + notarization + stapling pipeline works end-to-end** in CI on a fresh runner with a smoke Tauri app that contains the FFmpeg sidecar. Per RESEARCH.md and PITFALLS.md §2, this is the #1 release blocker — if this CI is not green by end of Wave 0, every downstream plan is at risk.

Purpose: D-22 locks static-universal FFmpeg. D-41 locks Developer ID signing + hardened runtime + notarytool. Tauri issues #8075 and #11992 document that unsigned nested Mach-Os are the single most common cause of late-stage release failure. This plan eliminates that risk in Wave 0 before any domain code is written.

Output: `scripts/build-ffmpeg/` recipes, `scripts/notarize/` pipeline, a minimal smoke Tauri app used only for notarization validation, and two CI workflows (`ffmpeg-build.yml` + `notarize-smoke.yml`) that run on every PR touching these paths and must stay green.
</objective>
