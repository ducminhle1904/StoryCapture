    pub width_px: u32,          // physical pixels (retina-corrected)

  <behavior>
    - `ByteBoundedQueue` is an mpsc-style channel with a `cap_bytes: usize` limit (default 256 MiB per D-19). `try_push(frame) -> Result<(), DroppedFrame>`: computes `frame.byte_size()`; if `current_bytes + size > cap_bytes`, drops the NEW frame, increments `stats.dropped_frames`, logs at WARN via tracing. Receiver decrements `current_bytes` on consume. Use a `tokio::sync::Mutex<VecDeque<Frame>>` + `tokio::sync::Notify` for simplicity; OR use `tokio::sync::mpsc::channel` wrapped with a size accumulator.
    - `Pts` struct: `{ ns: i128, source: ClockSource }` where `ClockSource` is `HostTime` (macOS CMTime→host time) or `Qpc` (Windows QueryPerformanceCounter). NEVER rewrite timestamps; pass through from capture API.
    - `DisplayInfo`: `{ id: DisplayId, name: String, width_px: u32, height_px: u32, scale_factor: f32, is_primary: bool }`. `width_px`/`height_px` are physical pixels (retina-corrected). `scale_factor` = backingScaleFactor on mac / DPI/96 on win.
    - `SckBackend` uses `screencapturekit` crate pinned `=1.70.0`. Config: `SCStreamConfiguration.width = display.width * scale`, `.height = display.height * scale`, `.pixelFormat = kCVPixelFormatType_32BGRA` (or NV12 if requested), `.showsCursor = cfg.include_cursor`, `.queueDepth = 8` (SCK-internal; our downstream queue is byte-bounded).
    - TCC flow in `tcc.rs`: `pub fn preflight_screen_capture_access() -> PermissionState` returns `Granted | Denied | Undetermined` via FFI to `CGPreflightScreenCaptureAccess()`. On Denied, `pub fn request_and_guide(app: &AppHandle)` opens `x-apple.systempreferences:com.apple.preference.security?Privacy_ScreenCapture` via `tauri_plugin_opener`, shows a modal (emitted as event `capture:tcc-denied` for UI to render), then exposes `pub fn relaunch_after_grant()` which uses `tauri::process::restart` or `Command::new("open").args(["-a", bundle_id])`.
