<objective>
Build the encoder crate that owns the FFmpeg sidecar lifecycle: spawn the static binary from Plan 02, probe hardware encoders at startup (VideoToolbox / NVENC / QSV / AMF → libopenh264 fallback → error), pipe `capture::Frame` bytes into FFmpeg stdin, parse progress from stderr, write a signed MP4/H.264 file. Add an `ffprobe`-based A/V alignment CI check that fails on >100 ms drift across a synthetic 10-minute recording (D-26, ENC-05).

Purpose: This crate is where "frames captured" becomes "playable MP4" — the final piece of the Phase 1 end-to-end pipeline. A/V drift is the silent killer (PITFALLS.md §6); catching it in CI prevents shipped-broken videos. HW-encoder probing prevents "works on my machine" where developer has NVENC but user doesn't.

Output: Working `encoder` crate, FFmpeg progress parser, drift-check CI workflow, Tauri host command for recording orchestration.
</objective>

    - For Phase 1, every recording uses H.264 in MP4 at capture-native resolution and capture-native framerate (VFR passthrough via `-vsync vfr`).     - FFmpeg command line (generated by `EncodeConfig::to_ffmpeg_args`):
      ```
      -hide_banner -y
      -f rawvideo -pix_fmt bgra -s <W>x<H> -r <fps_advisory> -use_wallclock_as_timestamps 0 -i pipe:0
      -f lavfi -i anullsrc=r=48000:cl=mono
      -c:v <encoder> -b:v 12M -profile:v baseline -level 4.1 -pix_fmt yuv420p
      -c:a aac -b:a 64k
      -vsync vfr
      -movflags +faststart
      -progress pipe:2 -loglevel info
      <output_path>
      ```
    - Stdin pump: reads `Frame` from `mpsc::Receiver`, for `FrameData::Owned` writes bytes directly; for native handles (mac CVPixelBuffer / win D3DTexture) performs a CPU copy to BGRA bytes (since FFmpeg stdin is not zero-copy from native surfaces — true zero-copy would require VideoToolbox/NVENC API integration not a subprocess — document this). Phase 1 accepts this copy cost; Plan 11 or Phase 2 optimize. The capture crate's `Frame::into_bgra_bytes()` helper (add to Plan 07's Frame impl OR implement here via trait) produces contiguous BGRA bytes respecting stride.
    - Progress parser: FFmpeg with `-progress pipe:2` emits key=value lines to stderr every ~1s. Parse `frame=`, `fps=`, `bitrate=`, `out_time_ms=`, `drop_frames=`, `dup_frames=`, `speed=`. Emit `EncodeProgress { frames, fps, out_time_ms, speed, bitrate_kbps }` via `mpsc::Sender<EncodeProgress>`.
    - Graceful shutdown: on stop, drop stdin (sends EOF), await child exit with 15s timeout; on timeout, SIGTERM then SIGKILL. Do NOT orphan the process.
    - Error handling: FFmpeg stderr tail (last 2 KiB) captured on non-zero exit and wrapped in `EncoderError::FfmpegExit { code, stderr_tail }` for UI.

    - Spawns FFmpeg with args
    - Spawns frame-pump task: loops `while let Some(frame) = frames.recv().await`, converts to contiguous BGRA bytes, `stdin.write_all(&bytes).await?`; on recv close, drops stdin (EOF)
    - Spawns progress-parser task: reads stderr → parses → forwards to `progress_tx`
    - Awaits child exit; collects stderr tail on failure
    - Returns `EncodeResult { output_path, duration_ms, bytes, frames_written, frames_dropped }`
    - Handles stdin write errors: if FFmpeg exits early (disk full, invalid args), write fails with BrokenPipe → stop frame pump, log ERROR, collect stderr, return `FfmpegExit`

    **`scripts/ci/check-av-drift.sh`**:
    ```bash
    #!/usr/bin/env bash
    set -euo pipefail
    FFPROBE=${FFPROBE:-./ffprobe}
    INPUT=${1:?"usage: $0 <input.mp4>"}
    MAX_DRIFT_MS=100

    video_end_s=$("$FFPROBE" -v error -select_streams v:0 -show_entries stream=duration -of csv=p=0 "$INPUT")
    audio_end_s=$("$FFPROBE" -v error -select_streams a:0 -show_entries stream=duration -of csv=p=0 "$INPUT")
    diff_ms=$(awk -v v="$video_end_s" -v a="$audio_end_s" 'BEGIN { d = (v - a) * 1000; if (d < 0) d = -d; printf "%.0f\n", d }')

    echo "video_end=${video_end_s}s audio_end=${audio_end_s}s diff=${diff_ms}ms"

    if [ "$diff_ms" -gt "$MAX_DRIFT_MS" ]; then
      echo "ERROR: A/V drift ${diff_ms}ms exceeds ${MAX_DRIFT_MS}ms threshold"
      exit 1
    fi
    echo "OK: A/V drift ${diff_ms}ms within ${MAX_DRIFT_MS}ms"
    ```
