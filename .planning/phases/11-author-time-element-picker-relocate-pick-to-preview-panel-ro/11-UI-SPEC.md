## Copywriting Contract

All strings locked. Tone: concise, neutral, sentence case, no terminal period on single-phrase labels. English only (matches Phase 10; Vietnamese strings are legacy to `DryRunPanel` only).

### Pick button — visible label + aria-label

| Context | Visible | aria-label |
|---------|---------|------------|
| Default | Icon-only (`Crosshair` 14px) | `Pick element from preview (Cmd-Shift-P)` |
| Hover / focus | Icon + `⌘⇧P` kbd pill appears inline | `Pick element from preview (Cmd-Shift-P)` |
| Active (picking) | Icon (filled variant) + `Esc` pill replaces `⌘⇧P` | `Picking — press Esc to cancel` + `aria-pressed="true"` |
| Starting | `Loader2` spinner (animate-spin) + text "starting…" | `Starting author session…` |
| Disabled | Icon (muted) | inherits per-state tooltip as `aria-describedby` |

**Rationale for icon-first default:** Preview toolbar is dense (scene selector + viewport + quality controls per Phase 10). An icon-only button keeps horizontal budget small. The `⌘⇧P` pill becomes inline on hover/focus — discoverable without stealing permanent width.

### Per-state tooltips (Base UI `Tooltip`, 12px, 150ms open delay)

| FSM state (D-16) | Pick button tooltip |
|------------------|---------------------|
| `Idle` (author-session dormant) | **Pick element · starts Preview** |
| `LivePreview` | **Pick element · ⌘⇧P** |
| `Picking` | **Picking — press Esc** |
| `Picking` (UI-local `starting…`) | **Starting session…** |
| `SimulatorRunning` | **Simulator running — cancel to pick** |
| `SimulatorPaused` | **Paused at step {N} — Pick will resume Preview after** |

`{N}` is the 1-indexed paused ordinal sourced from `simulatorStore.pausedAtOrdinal` (Phase 10-03). If ordinal is unavailable, the tooltip omits the phrase: `Paused — Pick will resume Preview after`.

### Picking banner (relocated into the Preview panel)

| Variant | Copy |
|---------|------|
| Active pick | **PICKING — press Esc to cancel** |
| Active pick from `SimulatorPaused` | **PICKING — press Esc (Preview will stay paused at step {N})** |
| Internal error (session warm-up failed, sidecar offline, picker-start rejected) | **Couldn't start picker — {short reason}. Try again or toggle Preview off and on.** |

Banner uses `aria-live="polite"` by default (matches Phase 7 baseline). The error variant escalates to `aria-live="assertive"` + `role="alert"`.

### Toasts (Sonner — global toast provider)

| Event | Copy |
|-------|------|
| Pick resolved (first pick on line — step-id stamped) | **Added `{verb} {target}` · line {L}** |
| Pick resolved (re-pick — targets.json-only update) | **Updated fallback for step {N}** |
| Pick cancelled by Esc | No toast (silent — consistent with Phase 7) |
| Pick cancelled by navigation | **Picking cancelled — page navigated** |
| Pick cancelled by unsupported URL | **Picking unavailable on this page (`chrome://`, `about:`, or `view-source:`)** |
| Warm-up replay partial failure | **Warming context hit an error — picking on whichever page loaded** |
| Timeout (60 s with no resolution) | **Picker timed out — try again** |

### Record-path primary-miss (HUD + error toast — D-06)

Exact copy, single source of truth, quoted verbatim by the Executor error variant and rendered inside `recorder/hud.tsx`:

> **Step {N}: `{verb} {target}` could not match any element.**
> **Self-healing is disabled during recording. Open this story in Simulator, use "Promote to fallback" on step {N}, then try again.**

Surfaced as:
- **HUD panel:** 2-line block, destructive left border, inline link **"Open in Simulator"** (right-aligned, 12px, accent color, underline-on-hover). Click → opens the Editor on the failed step (scrolls to step `{N}`, sets caret on that line, focuses CodeMirror). Phase 11 does **not** auto-start the simulator — the user decides.
- **Toast (shadow):** 13px body with "Open in Simulator" action button (Sonner `action` slot). Click behaves identically to the HUD link. Toast auto-dismisses at 12 s.

No banner variant for this error — the Record path has no picker banner after Phase 11 (the banner lives in Preview, which is not open during a record).

### Static copy spec table (for the checker)

| Element | Copy |
|---------|------|
| Primary CTA | Pick element · ⌘⇧P |
| Empty state heading | Pick element from preview |
| Empty state body | Use Pick to let Story author-time insert an accessible DSL line that uniquely targets an element in your running app. Toggle Preview on to begin. |
| Error state | Couldn't start picker — {short reason}. Try again or toggle Preview off and on. |
| Destructive confirmation | No destructive confirmation in Phase 11 (no destructive actions; re-pick is additive — demotes prior primary to `fallbacks[0]`). |

## Visual Layout & Interaction

### 1. PreviewPickerButton.tsx

**Shape:** icon-first, `Base UI Button` `variant="ghost"` `size="sm"`, aligned with the Preview toolbar's existing control cadence.

**Mount location (in `preview-panel.tsx` header):**

```
┌── [scene label] ─── [PreviewPickerButton] [viewport selector] [quality selector] ─┐
```

Pick sits **left** of the viewport/quality controls (per CONTEXT Specific Ideas). `gap-2` between controls.

| Region | Spec |
|--------|------|
| Button container | `inline-flex items-center gap-1 h-7 px-2.5 rounded-[var(--radius-sm)] border border-transparent bg-[var(--color-surface-200)] text-[var(--color-fg-primary)] transition-colors` |
| Hover | `hover:bg-[var(--color-surface-300)]` (idle + LivePreview only) |
| Active (picking) | 1px `border-[var(--color-accent-primary)]`; icon swaps to `Crosshair` **filled** variant; background remains surface-200 (no accent fill — accent is decorative, per CLAUDE.md) |
| Starting | `Loader2` 14px with `animate-spin` **OR** static loader under `prefers-reduced-motion`; inline text "starting…" appears in 12px muted |
| Disabled | `opacity-60 cursor-not-allowed hover:bg-[var(--color-surface-200)]` |
| Icon size | 14px (lucide) |
| Kbd-hint pill | inline after icon, 10px monospace, `px-1 py-[1px] rounded-[4px] bg-[var(--color-surface-300)] text-[var(--color-fg-muted)]`; appears **on hover or `:focus-visible`**; replaced by `Esc` pill in active state |
| Focus ring | inherits global `:focus-visible` → 2px `--color-focus-ring` outline with 2px offset |

**Button states → FSM mapping (per D-16 + research A3 `starting…` is UI-local):**

| FSM state | `disabled` | Visual | Click action |
|-----------|-----------|--------|--------------|
| `Idle` | no | default | `picker_start_author(streamId, cursorLine)` (lazy-starts session via `start_author_preview` first) |
| `LivePreview` | no | default | `picker_start_author(streamId, cursorLine)` |
| `Picking` | no | active (accent border + filled icon + `Esc` pill) | call `pickElement.cancel` (click-to-cancel = identical to Esc) |
| `SimulatorRunning` | **yes** | disabled | no-op; tooltip only |
| `SimulatorPaused` | no | default | `picker_start_author(…)` with `resume_to=Paused` transition |
| UI-local `starting…` | **yes** (button blocks re-click during warm-up) | spinner + muted text | no-op |

Button re-click while `Picking` acts as cancel (toggles off). This is safer than dimming the button (user cannot get stuck) and is consistent with Phase 10's single-click-Cancel.

### 2. Picking banner (relocated into Preview panel surface)

**Position:** sticky at the top of the Preview panel's **content** area (below the toolbar row, above the viewport canvas). Not a global overlay — lives inside the Preview panel surface.

```
┌─ ⊕ PICKING — press Esc to cancel ──────────────────────── ┐
│ [ viewport canvas — author-session screencast PAUSED ]    │
```

| Dimension | Value |
|-----------|-------|
| Height | 32px fixed |
| Background | `bg-[var(--color-surface-300)]` (active pick, neutral); `bg-[var(--color-danger)]/12 border-[var(--color-danger)]/40` (error variant) |
| Left icon | `lucide-react` `Crosshair` 14px `--color-accent-primary` (active) / `AlertTriangle` 14px `--color-danger` (error) |
| Text | 13px 500 `--color-fg-primary`; `PICKING` uppercase tracking `[0.08em]`; suffix clauses (`— press Esc …`) `--color-fg-muted` |
| Right region | none (no Cancel button — Esc + button re-click already cover cancel; matches Phase 7 minimalism) |
| Animation | Fade-in/out 120ms via `motion/react` opacity 0↔1; respects `prefers-reduced-motion` (instant toggle) |
| ARIA | `role="status" aria-live="polite"` (active); `role="alert" aria-live="assertive"` (error variant) |
| Coexistence with Phase 10 "Simulator running" banner | Two banners never coexist — `Picking` and `SimulatorRunning` are mutually exclusive in D-16; `Picking{resume_to=Paused}` replaces the Phase 10 `Simulator paused …` banner for the duration of the pick |

### 3. `starting…` UI-local loading state

Triggered on first-Pick lazy-start (D-09). This is **not** an FSM state — it's a component-local flag (research A3: "UI-local, not FSM"). Flow:

1. User clicks Pick while `AuthorDriverState::Idle`.
2. Button enters local `isStarting=true`: swap icon → `Loader2` + append " starting…" in 12px muted. Button is `disabled` and `aria-busy="true"`.
3. Host begins: acquire registry lock → `start_author_preview` → navigate-only replay → transition to `Picking` state.
4. On success, FSM → `Picking`; UI sets `isStarting=false` and shows the active state (accent border + filled icon + Esc pill) + reveals banner.
5. On warm-up error (navigate failure, sidecar offline): FSM reverts to `Idle`; UI dismisses `isStarting`; banner error variant fires with the internal-error copy above.

**Timeout:** if `isStarting` exceeds **15 s** with no host response, UI force-clears the flag and fires an error toast ("Starting the session took too long — try again."). This is a UX safety net; does not pre-empt the host's own 60 s picker timeout.

### 4. Active-pick animation budget

Honor `prefers-reduced-motion: reduce` everywhere.

| Animation | Duration | Easing | Reduced-motion fallback |
|-----------|----------|--------|-------------------------|
| Banner fade-in/out | 120ms | `easeOut` / `easeIn` | instant toggle |
| `Loader2` spinner | 1s linear rotation (lucide default) | linear | static icon (no rotation) |
| Pick button active-state border | instant | — | — |
| Crosshair-icon filled↔outline swap | instant | — | — |
| Kbd-hint pill fade-in on hover/focus | 80ms | `easeOut` | instant |

**No pulse, no bounce, no "breathing" treatment on the active button** — Phase 10 establishes instant signals for state transitions; Phase 11 preserves that register.

### 5. Record-path primary-miss HUD + toast

Lives inside `apps/desktop/src/features/recorder/hud.tsx`. The Record toolbar no longer shows Pick (D-05), so the HUD gains the sole responsibility for surfacing this failure mode.

```
┌─ ⚠ Step 7: click "Save" could not match any element. ───── Open in Simulator → ┐
│   Self-healing is disabled during recording. Open this story in Simulator, use   │
│   "Promote to fallback" on step 7, then try again.                               │
```

| Dimension | Value |
|-----------|-------|
| Container | `rounded-[var(--radius-md)] border-l-4 border-[var(--color-danger)] bg-[var(--color-surface-200)] px-3 py-2 flex gap-3` |
| Left icon | lucide `AlertTriangle` 16px `--color-danger` |
| Heading line | 13px 500; verb excerpt (`click "Save"`) in 11px mono `--color-fg-primary` on `--color-surface-300/60` pill |
| Body | 12px 400 `--color-fg-muted`; includes step number inline (same `{N}` as heading) |
| Action link | `Open in Simulator →` — right-aligned, 12px 500, `--color-accent-primary`, underline-on-hover |
| Keyboard | HUD is focusable (`tabIndex=0`); `Enter` on the action link = click; `Esc` dismisses the HUD (same as other recorder HUD entries) |
| Toast shadow | Sonner `variant="destructive"`, 12 s auto-dismiss, with `action` slot labelled `Open in Simulator` |

Both surfaces (HUD + toast) are emitted on the same event; user can act on either. This matches how the recorder already handles `StepFailed` events (confirmed by reading `hud.tsx` in research) and doesn't introduce a new error-surface pattern.

### 6. Cmd-Shift-P keymap registration

- Registered in `codemirror-setup.ts` **keymap** block, in the same section Phase 10-03 registers `Cmd-.`. Do not use a global `document.addEventListener('keydown')` (research "Don't Hand-Roll").
- When the CodeMirror editor has focus **and** `AuthorDriverState` is not `SimulatorRunning`: `Cmd-Shift-P` / `Ctrl-Shift-P` triggers the same action as a Pick button click.
- When the editor is focused during `SimulatorRunning`: the keybinding is a **no-op with subtle shake on the Simulator banner** (matches Phase 10's treatment of `Cmd-.` while editor is read-only).
- When the editor is focused during `Picking`: the keybinding cancels the active pick (symmetric to clicking the button again).

The button's `aria-keyshortcuts="Meta+Shift+KeyP Control+Shift+KeyP"` is set so screen-reader users discover the shortcut.

## Accessibility (WCAG 2.1 AA)

| Surface | Control |
|---------|---------|
| Pick button | `role="button"` (native via `<button>`), `type="button"`, `aria-label` per-state (see Copywriting), `aria-describedby={tooltipId}`, `aria-pressed="true"` during `Picking`, `aria-busy="true"` during `starting…`, `aria-keyshortcuts="Meta+Shift+KeyP Control+Shift+KeyP"`. Enter and Space both activate (native `<button>` behavior). |
| Banner (active pick) | `role="status" aria-live="polite" aria-atomic="true"` |
| Banner (error) | `role="alert" aria-live="assertive"` |
| Tooltip | Base UI `Tooltip` with `aria-describedby` wiring already provided by the primitive; 150ms open delay; disappears on `Escape` |
| Kbd-hint pill | `aria-hidden="true"` (decorative only; true shortcut announced via `aria-keyshortcuts`) |
| Record-path HUD | `role="alertdialog"` unnecessary (non-modal); use `role="region" aria-labelledby` on the HUD container; inline "Open in Simulator" link is a native `<button>` with `aria-label="Open story in Simulator on step {N}"` |
| Record-path toast | Sonner default handles `role="status"` + auto-announce |
| Focus ring | all Phase 11 interactive controls inherit global `:focus-visible` → 2px `--color-focus-ring` with 2px offset (matches Phase 10) |
| Color is never sole signal | Pick button active state pairs accent border with a **filled icon variant** (shape change); disabled state pairs muted color with reduced opacity + cursor-not-allowed (affordance change); error toast pairs danger color with `AlertTriangle` icon |
| Keyboard trap | None. Tab order inside Preview panel: scene selector → PreviewPickerButton → viewport selector → quality selector → canvas. `Esc` inside `Picking` cancels the pick and returns focus to the Pick button. |
| Reduced-motion | `prefers-reduced-motion: reduce` disables `Loader2` spin, banner fade, and kbd-pill fade per §Active-pick animation budget |

**Contrast spot-checks (dark theme, target AA 4.5:1 for text <18px):**

- 13px 500 `--color-fg-primary` on `--color-surface-300` (banner): ≥7.0:1 ✓
- 12px 400 `--color-fg-muted` on `--color-surface-200` (button tooltip body): ~4.7:1 ✓ (same pair Phase 10 validated)
- 10px 600 `--color-fg-muted` on `--color-surface-300` (kbd-hint pill): ~5.1:1 ✓
- 12px 500 `--color-accent-primary` on `--color-surface-200` (Open-in-Simulator link): ~5.5:1 ✓

**Screen-reader narration flow during a typical pick:**

1. Focus Pick button → SR reads `"Pick element from preview. Shortcut Command Shift P."` (combined `aria-label` + `aria-keyshortcuts`).
2. Press Enter → button becomes `aria-pressed="true"`; banner announces `"Picking — press Esc to cancel."` via `aria-live="polite"`.
3. User clicks an element in the viewport → banner dismisses; focus returns to the Pick button; toast announces `"Added click Save button line 12."` (Sonner polite region).
4. If user presses Esc instead → banner dismisses silently (no toast per copy table); Pick button regains `aria-pressed="false"`.
