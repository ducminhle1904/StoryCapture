### 4. Preview panel mode swap

When `simulatorStore.runState === "running" | "paused" | "complete"` and `currentFrameOrdinal` is set:

- **Live canvas (9-04 `LivePreview` component)** is **hidden** (not dimmed вҖ” replaced cleanly to avoid flicker). Canvas remount is acceptable; CONTEXT does not require hot-swap.
- **Static frame**: `<img src={convertFileSrc(frame.screenshot_path)}>` renders at the same aspect ratio as the active viewport. Preserves the existing rounded-corner device-frame treatment inside `preview-panel.tsx`.
- **Bbox overlay**: if `frame.matched_bbox` present, absolute-position `<div>` border 2px `--color-accent-primary` inside the image.
- **Cursor dot**: 8px circle `bg-[var(--color-warning)]` with 1px black outline, at `cursor_xy` scaled to the image's actual render size.
- **Status chip at top-right of viewport frame**: `matched` (green) / `fuzzy` (warning) / `none` (danger) / `no selector` (info) вҖ” 10px uppercase pill matching existing viewport label style.
- **On `runState === "idle"`**: preview panel restores the 9-04 live canvas if `previewEnabled=true`, else falls back to the shipped `Globe` placeholder.

**Transition feel (Claude's discretion):** stepwise, not jarring. When the scrubber moves, crossfade the preview image 100ms via `motion/react` (`opacity 0.7 вҶ’ 1`). Bbox + cursor repaint instantly (no animation) to avoid "chasing" feel.

### 5. "Preview to here" caret context-menu + keyboard shortcut

- **Right-click on any DSL step line** вҶ’ context menu overlay with a single Phase 10 item:
  - Icon: lucide `PlayCircle` 14px
  - Label: `Preview to here`
  - Right-aligned keyboard hint: pill-shaped 10px mono `вҢҳ.` on `--color-surface-300` background
- **Keyboard shortcut** `Cmd-.` (macOS) / `Ctrl-.` (Windows) with caret on a step line: same action. Registered via CM6 `keymap` in `codemirror-setup.ts`.
- **Additional shortcuts (Claude's discretion):**
  - `Cmd-Shift-.` (macOS) / `Ctrl-Shift-.` (Win): run full simulator from start. Matches the `Cmd-.` family and mirrors Phase 3's `Cmd+Shift+D` idiom without colliding.
  - `вҶҗ` / `вҶ’` while timeline focused: step through frames one at a time.
  - `Home` / `End`: jump to first / last frame.
  - `Esc`: cancel active run (consistent with Cancel button, no hold).
- **When editor is read-only** (during active run): context menu item shows the same label but with a small muted suffix "вҖ” run in progress" and is disabled; `Cmd-.` is a no-op with a subtle shake animation on the banner.

### 6. Promote-to-fallback button

Rendered **inline inside the filmstrip frame card** for frames where `matched_selector` was a fuzzy match (warning state). Tucked into the frame's bottom-right corner as a compact icon-only `Base UI Button` ghost variant, 16Г—16px, lucide `ArrowUpRight`. Tooltip: "Promote this match to a fallback selector".

On click:
1. Optimistic UI: button flips to `CheckCircle2` icon for 1.2s.
2. Dispatches `simulator_promote_fallback(run_id, ordinal)` вҖ” which runs the Phase 7-04c self-healing protocol explicitly.
3. Sonner toast "Fallback added to .story.targets.json for step N." OR error toast on failure.
4. Frame border shifts from warning dotted вҶ’ solid success 1px (signals promoted state in the current run's view).
5. Action is idempotent per frame within a run; button hides after first click.

**Never auto-promote** (D-07). Never bulk-promote (no "promote all fuzzy" CTA вҖ” each fuzzy match is an explicit per-step decision).

### 7. Error-state frame

When a step fails mid-run, the frame card:
- Border: 2px solid `--color-danger`.
- Top-right overlay: 14px `AlertTriangle` icon `--color-danger`.
- Inline below the filmstrip (when this frame is selected): red message bar `text-[11px] text-[var(--color-danger)]` with copy **"Step 7: selector not found вҖ” `getByRole('button', { name: 'Save' })`"**. Copy-button on the selector text (lucide `Copy`, 12px).
- Banner updates to failed-state variant (see В§2).
- Filmstrip **stops advancing** (no new frames after the failed ordinal); scrubber remains at the failed ordinal.
- Timeline still scrubable across the frames captured before failure.
