# StoryCapture - Conventions

Concrete patterns used in the current Electron/TypeScript codebase.

## TypeScript / React

- File naming: kebab-case for new files. PascalCase is reserved for existing
  legacy/dialog components already in that style.
- Desktop feature code lives under `apps/desktop/src/features/<feature>/`.
- Shared renderer UI lives under `apps/desktop/src/components`.
- Host IPC facades live under `apps/desktop/src/ipc`.
- Electron host code lives under `apps/desktop/electron`.
- Cross-feature renderer state lives under `apps/desktop/src/state` or
  `apps/desktop/src/stores`.
- Web app code lives under `apps/web/src`; App Router pages/routes live under
  `apps/web/src/app`, server logic under `src/trpc` and `src/lib`.

## State

- Prefer one feature-owned Zustand store per feature.
- Use slice composition only where the feature is already split that way, such
  as the post-production editor store.
- TanStack Query wrappers should stay in `src/ipc/*.ts` with stable query-key
  factories and explicit invalidation.
- Keep server/host state outside presentational components when a local IPC or
  tRPC wrapper already exists.

## IPC

- Renderer code may continue using `@tauri-apps/api` and plugin packages while
  the compatibility layer exists.
- New host behavior belongs in `apps/desktop/electron/ipc/*`, registered
  through `ipc/handlers.ts` and dispatched by `ipc.ts`.
- `apps/desktop/electron/ipc.ts` is the bridge entrypoint, not the place to
  concentrate new host logic.
- Most current grouped modules are command ownership registries around
  `legacyHandlers([...])`; inspect the module and the matching `ipc/legacy/*`
  implementation before assuming logic is local to the grouped file.
- Prefer splitting new host commands into a focused grouped module with real
  implementation instead of expanding `ipc/legacy.ts`, unless the touched
  behavior is already tightly coupled to legacy state.
- Long-running operations should use existing channel/event shim patterns.
- Long-running native capture helpers stay behind typed backend/protocol
  adapters and the existing lifecycle/channel surfaces; do not expose an
  untyped helper protocol directly to renderer code.
- Keep renderer IPC facades thin and typed. Match existing argument casing and
  DTO conversions.

## Packages And Generated Files

- `packages/story-dsl` is editor/AST support. `src/ast.ts` is checked in but
  generated by `ts-rs`. Do not assume this package is the runtime parser.
- `packages/shared-types/src/ipc.ts` is the checked-in host IPC compatibility
  surface. It includes generated-style command wrappers and types for the
  Tauri-compatible invoke bridge.
- `packages/shared-types/src/recording-v2.ts` is the hand-maintained JSON-safe
  Recording V2 contract. It is public from the root barrel and the
  `@storycapture/shared-types/recording-v2` runtime-safe subpath.
- `packages/shared-types/src/generated/effects.ts` is generated by `ts-rs`.
  Do not hand-edit it unless you also update the generation source/provenance.
  It is not currently exposed through the `@storycapture/shared-types` package
  export map or root barrel.
- `packages/shared-types` currently has no package scripts; verify through
  consumers or root typecheck.

## UI

- Astryx is the generic component and visual-style foundation. Import generic
  components directly from `@astryxdesign/core/<Component>`; do not add local
  wrappers that only rename Astryx props.
- `@storycapture/ui` owns `StoryCaptureThemeProvider`, the compiled Gothic
  theme, local font CSS, and `--story-*` product tokens. The application theme
  is permanently dark and has no user-facing mode switch or persistence key.
- Icons come from `lucide-react`.
- Motion uses `motion/react`.
- Tailwind is restricted to layout, responsive behavior, position, overflow,
  and dimensions. Colors, typography, borders, radii, shadows, and interactive
  states come from Astryx components or theme tokens.
- Product-specific compound components are allowed only when they own behavior
  Astryx does not, such as Electron window chrome and `StoryColorField`.
- Recorded media, user-authored composition, and the export compositor remain
  outside the application theme boundary.
- Run `pnpm ui:legacy-check` after UI work. It rejects retired UI imports,
  dependencies, tokens, light-theme selectors, and local primitive directories.

## Web

- tRPC procedures should enforce workspace membership/RBAC through shared guard
  helpers before reading or mutating workspace-scoped data.
- Protected tRPC procedures currently authenticate through the Auth.js session
  in tRPC context. Desktop/SSE JWTs are verified explicitly in the routes or
  subscription procedures that accept them; do not assume generic bearer JWT
  support in protected procedures.
- Prisma schema changes must consider migrations, generated client output under
  `apps/web/src/generated/prisma`, seed data, and tests.
- API routes handling public ingest/upload/cron paths must document auth and
  rate/security assumptions in code or credentials docs.
- Desktop sync is metadata sync, not full project-file replication.

## Testing

Use the narrowest meaningful check for the change.

Known commands:

- Root typecheck: `pnpm typecheck`.
- Root lint: `pnpm lint`.
- Desktop tests: `pnpm --dir apps/desktop exec vitest run`.
- Desktop targeted test: `pnpm --dir apps/desktop exec vitest run <path>`.
- Desktop build/package: `pnpm --dir apps/desktop run build`.
- Web tests: `pnpm --dir apps/web test`.
- Web targeted test: `pnpm --dir apps/web exec vitest run <path>`.
- Web typecheck: `pnpm --dir apps/web typecheck`.
- UI tests: `pnpm --dir packages/ui test`.
- UI targeted test: `pnpm --dir packages/ui exec vitest run <path>`.
- Story DSL typecheck: `pnpm --dir packages/story-dsl typecheck`.

There is no root `pnpm test`. There is no current Playwright/E2E test command or
config; Playwright references in source are runtime automation concepts, not
test wiring.

Coverage shape:

- Desktop has many Vitest tests around editor, recorder, post-production UI and
  state, plus Electron permission and IPC helpers.
- Web tests currently cover workflow helpers, sync metadata, and template
  metadata. Auth, upload APIs, invite/RBAC, analytics cron, watch/embed/oEmbed,
  and route availability are not broadly covered.
- UI foundation tests live under `packages/ui/src/theme/` and cover the Gothic
  provider plus representative Astryx primitives and reset-cascade behavior.
- `packages/story-dsl` currently has no test files; use package typecheck plus
  consuming desktop tests for DSL/editor behavior.

When runtime behavior changes, run package tests plus a build or manual runtime
verification appropriate to the risk. Electron capture/signing/share flows may
still require operator UAT.

## Commits

- Format: `type(scope): subject`.
- Common types: `feat`, `fix`, `refactor`, `docs`, `chore`, `test`, `merge`.
- No `Co-Authored-By` trailers.
- Do not use `--no-verify`, `@ts-ignore`, skipped tests, or lint suppression to
  force checks through.

## Lint / Format

- Biome is the single repo formatter/linter.
- Config: `biome.json`.
- Run `pnpm lint` and `pnpm format` as needed.

## CI

The primary workflow is `.github/workflows/ci.yml`. It runs on pull requests
and pushes to `main`, installs pnpm/Node through `.github/actions/setup-toolchain`,
typechecks, runs desktop/UI/web tests, and builds the Electron desktop package.
There is no separate release or benchmark workflow in the current tree.

## Agent / Contributor Rules

1. Fix root causes; avoid workaround-only patches.
2. Keep edits scoped to the task.
3. Match established local patterns.
4. Read immediate callers, tests, and wrappers before editing behavior.
5. Do not revert user changes.
6. There is no `.planning/` directory in the current checkout. If planning docs
   are restored, treat phase/research docs as historical unless a current state
   file explicitly says otherwise.
7. Report skipped checks and residual risk clearly.
