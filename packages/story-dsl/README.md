# @storycapture/story-dsl

Story DSL AST/vocabulary and CodeMirror language support.

## What Lives Here

- `src/ast.ts`: checked-in Story AST/type surface historically generated by
  `ts-rs`; do not hand-edit it. Its Rust source and `cargo test -p story-parser`
  regeneration path were removed with the legacy Rust runtime, so the current
  checkout cannot regenerate this artifact.
- `src/codemirror-lang.ts`: CodeMirror stream language/token/highlight support.
  The package depends on CodeMirror autocomplete today, but it does not expose
  a standalone completion API.
- `src/index.ts`: package barrel.

## Important Boundary

This package is not the runtime automation parser. Runtime parsing is reached
from desktop renderer code through `apps/desktop/src/ipc/parse.ts` and handled
by `apps/desktop/electron/ipc/story-parser.ts`. Until a replacement generator is
introduced, new runtime command shapes belong in those TypeScript contracts;
`src/ast.ts` remains a historical generated surface rather than a file to patch
manually.

When changing DSL semantics, inspect both:

- editor support in this package and `apps/desktop/src/features/editor`;
- runtime parse/simulator/automation behavior in desktop IPC and Electron host
  code.
